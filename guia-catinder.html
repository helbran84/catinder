Flujo de autenticacion

  1. El usuario registra/inicia sesion desde el frontend
  2. El frontend llama directamente a supabase.auth.signUp() o signInWithPassword()
  3. Supabase retorna un JWT token (guardado en localStorage)
  4. El frontend usa ese token para llamar al backend (header Authorization: Bearer <token>)
  5. El backend decodifica el JWT directamente (sin llamar a Supabase) y busca el perfil en la tabla users

4. Todas las Rutas del Backend

Auth (/api/auth)

MetodoRutaAuthQue hace
POST/api/auth/registerNoCrear usuario: valida email @cat.com, crea en Supabase Auth + tabla users
POST/api/auth/loginNoLogin: autentica contra Supabase Auth, retorna JWT + perfil
POST/api/auth/sync-userNoSincronizar sesion: toma token Supabase, decodifica, retorna perfil
POST/api/auth/create-profileNoCrear perfil: inserta en tabla users despues de registro en Supabase Auth

Usuarios (/api/users)

MetodoRutaQue hace
GET/api/users/profileObtener perfil completo con URL firmada de foto
PUT/api/users/profileActualizar perfil (nombre, edad, campana, turno, piso, bio, intereses...)
POST/api/users/upload-photoSubir foto a Supabase Storage, actualizar users.photo
GET/api/users/discoverDiscover: perfiles paginados (50/pagina) con filtros
POST/api/users/swipe/:dir/:userIdSwipe like/nope. Si es mutuo → crear match
POST/api/users/break/availableActivar zona de descanso (20 min)
POST/api/users/break/stopDesactivar zona de descanso
GET/api/users/break/findBuscar personas disponibles en break
GET/api/users/break/zonesZonas agrupadas por campana
POST/api/users/break/invite/:userIdInvitar persona en break (crea match)

Matches (/api/matches)

MetodoRutaQue hace
GET/api/matchesTodos los matches del usuario con info del otro + ultimo mensaje
GET/api/matches/:id/messagesMensajes de un match (marca como leidos)
POST/api/matches/:id/messagesEnviar mensaje

Admin (/api/admin)

MetodoRutaQue hace
GET/api/admin/statsEstadisticas: total usuarios, activos, matches, mensajes
GET/api/admin/usersTodos los usuarios
PUT/api/admin/users/:id/toggle-statusActivar/desactivar usuario
PUT/api/admin/users/:id/toggle-adminToggle permisos admin
DELETE/api/admin/users/:idEliminar usuario completo (cascada)
