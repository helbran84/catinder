Guia Completa del Proyecto

Matching para la Empresa

1. Que es CATINDER?

CATINDER es una aplicacion tipo Tinder para que los empleados de la empresa se conozcan entre si. Los usuarios crean un perfil, hacen swipe a las personas que les interesan, y si hay match (los dos se dieron like), pueden chatear.

La app esta disenada para dos sectores: Cobranzas y Super Linea, con roles de Agente, Lider y Supervisor.

6. Funcionalidades

6.1 Autenticacion

  • Registro en 3 pasos: Datos personales, Turno/Empresa, Ubicacion
  • Login con email @cat.com y password
  • JWT token con expiracion de 7 dias
  • Solo emails corporativos validos (configurable via ALLOWED_DOMAINS)

6.2 Discover (Swipe)

  • Swipe derecha = like, izquierda = pasar
  • Filtros: buscar por nombre, filtrar por rol (Agente/Lider/Supervisor)
  • Muestra nombre, piso, turno, rol y bio
  • Animaciones CSS para swipe

6.3 Matches y Chat

  • Lista de matches mutuos
  • Chat en tiempo real con Socket.IO
  • Mensajes persistidos en Supabase
  • Indicador de usuario en linea

6.4 Break Zone

  • Zona de descanso por sector (Cobranzas / Super Linea)
  • Ve quielen estan disponibles en su turno
  • Separados por campana

6.5 Admin

  • Ver todos los usuarios
  • Crear, editar, eliminar usuarios
  • Desactivar cuentas
  • Badges de colores por rol

9. Deploy

9.1 Frontend (Vercel)

  1. Conectar repo de GitHub en vercel.com
  2. Configurar VITE_API_URL apuntando al backend
  3. Desactivar Deployment Protection en Settings
  4. Deploy automatico con cada push a main

9.2 Backend (Render)

  1. Crear Web Service en render.com
  2. Conectar repo GitHub
  3. Root Directory: backend
  4. Build: npm install
  5. Start: node server.js
  6. Agregar todas las env vars
  7. Plan: Free

9.3 Base de Datos (Supabase)

  1. Crear proyecto en supabase.com
  2. Crear tabla users con los campos necesarios
  3. Configurar Auth con email confirmado automaticamente
  4. Crear bucket profile-pictures en Storage
  5. Obtener URL, anon key y service role key

11. Decisiones Tecnicas

Por que raw HTTPS en vez de @supabase/supabase-js?
El cliente de Supabase JS retorna resultados vacios en contexto servidor. Se uso https nativo de Node.js para hacer requests directos al REST API de Supabase, lo cual funciona correctamente.
Por que vercel.json con rewrite?
El patron /((?!.*\.).*) permite que las rutas SPA funcionen (ej: /discover → index.html) mientras que los archivos estaticos (/logotipo.png) se sirven normalmente.
Por que ALLOWED_DOMAINS?
Para restringir el registro solo a emails corporativos. Se configura en Render y se valida en el endpoint /api/auth/register.
